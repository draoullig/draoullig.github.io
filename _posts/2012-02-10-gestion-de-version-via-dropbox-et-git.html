Lorsque l'on développe même à titre personnelle il est parfois intéressant de bénéficier d'une gestion de version pour structurer ses petits projets.

Pour cela j'utilise Git, pour sa simplicité de mise en œuvre et sa puissance. Si vous ne connaissez pas, je vous recommande la lecture de la documentation suivante : Git Community Book

J'avais besoin de pouvoir accéder à mes projets depuis plusieurs machines, j'ai donc mis mes sources dans le cloud en utilisant un service de partage de fichier en l'occurence Dropbox parce qu'il est multi-platforme et offre un espace de 2Go gratuit ce qui me suffit amplement.

Mise en oeuvre

On suppose que le nom du projet est Projet et que son répertoire associé porte le même nom. J'utilise un système Linux.

  1. Initialisation du projet via Git
    • ~/Projet $ git init
    • ~/Projet $ git add .
    • ~/Projet $ git commit -m "Premier commit"
  2. Création sous Dropbox en local
    • ~/Projet $ cd ~/Dropbox
    • ~/Dropbox $ mkdir Git
    • ~/Dropbox $ cd Git
    • ~/Dropbox/Git $ mkdir Projet.git
    • ~/Dropbox/Git $ cd Projet.git
    • ~/Dropbox/Git $ git init --bare
  3. Synchronisation des deux répertoires locaux
    • ~/Dropbox/Git $ cd Projet
    • ~/Projet $ git remote add origin ~/Dropbox/Git/Projet.git
    • ~/Projet $ git push origin master
  4. Rapatriement depuis une autre machine et mise à jour du dépôt sous Dropbox
    • ~/Répertoire destination $ git clone ~/Dropbox/Git/Projet.git
    • ~/Répertoire destination $ cd Projet
    • Modifier un fichier dans le répertoire Projet
    • ~/Projet $ git add nom du fichier modifié
    • ~/Projet $ git commit -m "Essai commit"
    • ~/Projet $ git push ~/Dropbox/Git/Projet.git
